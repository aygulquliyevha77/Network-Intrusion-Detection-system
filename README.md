# 🛡️ Network Intrusion Detection System (NIDS) Lab – Suricata
This project demonstrates the setup and configuration of a **Network Intrusion Detection System (NIDS)** using **Suricata** in a Linux virtual machine environment.The purpose of this lab is to understand how intrusion detection systems monitor network traffic, detect suspicious activities, and generate alerts for potential cyber attacks.
📌 This project was completed during my **Cybersecurity Internship at CodeAlpha**.
# 🎯 Project Objectives
- 🔍 Set up a **Network Intrusion Detection System (NIDS)**
- ⚙️ Configure **custom detection rules**
- 📡 Monitor network traffic continuously
- 🚨 Detect suspicious or malicious activities
- 🧪 Simulate attacks to test the IDS
- 📊 Analyze alerts generated by the system
# 🧪 Lab Environment
| Component | Description |
|---|---
| Operating System | Kali Linux Virtual Machine |
| IDS Tool | Suricata |
| Target Service | SSH |

# ⚙️ Step 1 – Installing Suricata
Update system packages:
```bash
sudo apt update
```
Install Suricata:
```bash
sudo apt install suricata -y
```
Verify installation:
```bash
suricata -V
```
✅ This confirms that Suricata is installed successfully.
# 🔐 Step 2 – Enabling SSH Service
SSH was used as the target service to simulate authentication attacks.
Start SSH service:
```bash
sudo systemctl start ssh
```
Enable SSH at boot:
```bash
sudo systemctl enable ssh
```
Check SSH status:
```bash
sudo systemctl status ssh
```
Expected output:
```
Active: active (running)
# 🧠 Step 3 – Creating Custom Detection Rules
Custom rules allow Suricata to detect specific types of attacks.
Open the Suricata local rules file:
```bash
sudo nano /etc/suricata/rules/local.rules
```
Add the following rule:
```
alert tcp any any -> any 22 (msg:"SSH brute-force attempt detected"; sid:1000010; rev:1;)
```
### Rule Explanation
| Rule Component | Description |
|---|---|
| alert | Generate alert |
| tcp | Monitor TCP traffic |
| any any | Any source IP and port |
| any 22 | Traffic targeting SSH port |
| msg | Alert message |
| sid | Unique rule ID |
Save and exit the file.
# 🛰️ Step 4 – Running Suricata
Start Suricata and apply the custom rule:
```bash
sudo suricata -S /etc/suricata/rules/local.rules -i eth0
```
📡 Suricata now monitors network traffic and detects suspicious activity.
# 📊 Step 5 – Monitoring Alerts
To observe alerts in real time:
```bash
sudo tail -f /var/log/suricata/fast.log
```
This file displays alerts generated by Suricata.
For structured logs:
```bash
sudo tail -f /var/log/suricata/eve.json
```
These logs can later be used for **security analysis or visualization tools**.
# ⚔️ Step 6 – Simulating an SSH Brute Force Attack
To test the IDS detection capability, multiple login attempts were performed.
Example command:
```bash
ssh username@VM-IP
```
Incorrect passwords were entered multiple times.
🚨 This triggered Suricata alerts.
# 🚨 Example Alert
Example output from Suricata:

TCP connection attempt to port 22
✅ This confirms that the IDS successfully detected suspicious activity.
