# 🚨 Network Intrusion Detection System using Suricata
## 📌 Project Overview
As part of my **Cybersecurity Internship at CodeAlpha**, I built a **Network Intrusion Detection System (NIDS)** using Suricata to monitor network traffic and detect potential cyber attacks in real time.
This project demonstrates how security analysts monitor network activity and identify suspicious behavior using IDS technologies.
## 🛠️ Technologies Used
- 🐧 Linux (Ubuntu)
- 🛡️ Suricata IDS
- 🔎 Nmap
- 🌐 Network utilities (ping, ssh)
- 💻 Command Line Interface
## ⚙️ Project Implementation
During this project I performed the following steps:
### 1️⃣ Configured Suricata IDS
Installed and verified Suricata to monitor network traffic.
suricata --version
### 2️⃣ Created Custom Detection Rules
Configured custom rules to detect suspicious network activity.
Examples:
alert icmp any any -> any any (msg:"ICMP Ping Detected"; sid:1000001; rev:1;)
alert tcp any any -> any any (flags:S; msg:"TCP SYN Scan Detected"; sid:1000002; rev:1;)
alert tcp any any -> any 22 (msg:"SSH Brute Force";flow:to_server;flags:S; detection_filter:track by_src,count 2,second 20;sid:1000006; rev:1;)
### 3️⃣ Started Network Monitoring
sudo suricata -c /etc/suricata/suricata.yaml -i eth0
### 4️⃣ Real-time Alert Monitoring
tail -f /var/log/suricata/fast.log
## 🚨 Attack Simulations
To test the system I simulated several network activities.
### ICMP Ping Scan
ping 8.8.8.8
### TCP SYN Scan
nmap -sS <target_ip>
### SSH Login Attempts
ssh localhost
Suricata successfully detected these activities and generated alerts in real time.
## 📊 Log Analysis
Alerts generated by Suricata were monitored and analyzed using:
cat /var/log/suricata/fast.log
and
cat /var/log/suricata/eve.json
